# Harmonest Credential & Configuration Summary

## 📋 Overview

This document provides a complete overview of where to store different types of credentials and configuration data in the Harmonest system.

## 🔐 Credential Storage Structure

### 1. Backend Secrets (AWS Secrets Manager)
**🚨 NEVER expose to frontend - Backend Lambda functions only**

| Service | Secret Name | Description | Example Value |
|---------|-------------|-------------|---------------|
| **Guesty for Hosts** |
| | `harmonest/{env}/guestyforhosts/creds` | Login credentials | `{"email":"user@example.com","password":"secret"}` |
| | `harmonest/{env}/guestyforhosts/webSession` | Session cache | `{"utoken":"...","stoken":"...","cookies":"..."}` |
| **TTLock** |
| | `harmonest/{env}/ttlock/credentials` | API credentials | `{"username":"...","password":"...","app_id":"...","app_secret":"...","country_id":"67","site_id":"2"}` |
| | `harmonest/{env}/ttlock/token` | Access token storage | `{"access_token":"...","expires_at":1692216656.789}` |
| **QRLock** |
| | `harmonest/{env}/qrlock/credentials` | API credentials | `{"email":"user@example.com","password":"password"}` |
| | `harmonest/{env}/qrlock/token` | Access token storage | `{"access_token":"...","expires_at":1692209856.789}` |
| **Email Services** |
| | `harmonest/{env}/email/smtp-credentials` | SMTP settings | `{"host":"smtp.gmail.com","port":587,"username":"...","password":"..."}` |
| | `harmonest/{env}/email/api-keys` | Email API keys | `{"sendgrid":"SG.xxx","ses":"AKIA..."}` |
| **Payment** |
| | `harmonest/{env}/payment/stripe` | Stripe credentials | `{"publishableKey":"pk_...","secretKey":"sk_...","webhookSecret":"whsec_..."}` |
| | `harmonest/{env}/payment/paypal` | PayPal credentials | `{"clientId":"...","clientSecret":"..."}` |
| **External APIs** |
| | `harmonest/{env}/external-apis/google-maps` | Google Maps API | `{"apiKey":"server_key","publicApiKey":"frontend_key"}` |
| | `harmonest/{env}/external-apis/analytics` | Analytics keys | `{"googleAnalytics":"GA-...","mixpanel":"..."}` |
| | `harmonest/{env}/external-apis/sms` | SMS service | `{"twilio":{"accountSid":"...","authToken":"...","phoneNumber":"..."}}` |

## 🛠️ How to Use Each Type

### Backend Lambda Functions
```python
from common.credentials import (
    get_guesty_creds, get_guesty_session, update_guesty_session,
    get_ttlock_creds, get_ttlock_token, update_ttlock_token,
    get_qrlock_creds, get_qrlock_token, update_qrlock_token,
    get_stripe_creds, get_email_creds
)

# Get Guesty credentials
guesty_creds = get_guesty_creds()
email = guesty_creds['email']
password = guesty_creds['password']

# Get/update Guesty session
guesty_session = get_guesty_session()
utoken = guesty_session.get('utoken')
update_guesty_session({"utoken": "...", "stoken": "...", "cookies": "...", "exp": 1692216656})

# Get TTLock credentials
ttlock_creds = get_ttlock_creds()
username = ttlock_creds['username']
password = ttlock_creds['password']
app_id = ttlock_creds['app_id']
app_secret = ttlock_creds['app_secret']

# Get/update TTLock token
ttlock_token = get_ttlock_token()
access_token = ttlock_token.get('access_token')
update_ttlock_token({"access_token": "new_token", "expires_at": 1692216656.789})

# Get QRLock credentials
qrlock_creds = get_qrlock_creds()
email = qrlock_creds['email']
password = qrlock_creds['password']

# Get/update QRLock token
qrlock_token = get_qrlock_token()
access_token = qrlock_token.get('access_token')
update_qrlock_token({"access_token": "new_token", "expires_at": 1692209856.789})

# Get Stripe credentials
stripe_creds = get_stripe_creds()
secret_key = stripe_creds['secretKey']  # Never expose to frontend!

# Get email credentials
email_creds = get_email_creds()
smtp_host = email_creds['host']
smtp_username = email_creds['username']
```

## 🔑 Credential Types by Service

### Guesty for Hosts
- **Backend**: Email/password for API authentication
- **Frontend**: None (all API calls go through backend)

### TTLock
- **Backend**: Client ID, Client Secret, Access Token
- **Frontend**: None (all API calls go through backend)

### QRLock
- **Backend**: Email/password for API authentication
- **Frontend**: None (all API calls go through backend)

### Email Services
- **Backend**: SMTP credentials, API keys
- **Frontend**: None

### Payment (Stripe)
- **Backend**: Secret key, webhook secret
- **Frontend**: Publishable key only

### Google Maps
- **Backend**: Server-side API key (for geocoding, etc.)
- **Frontend**: Public API key (for map display)

### Analytics
- **Backend**: None typically
- **Frontend**: Public tracking IDs (Google Analytics, etc.)

## 🚀 Deployment Workflow

1. **Deploy Secrets Stack** → Creates empty secrets in AWS Secrets Manager
2. **Update Secrets** → Fill secrets with actual credentials using AWS CLI
3. **Deploy Application Stacks** → Lambda functions can access secrets
4. **Generate Frontend Config** → Create public configuration file
5. **Deploy Frontend** → Frontend uses public configuration

## 🔒 Security Best Practices

### ✅ DO
- Store all sensitive credentials in AWS Secrets Manager
- Use separate secrets for different environments (dev/prod)
- Use KMS encryption for all secrets
- Generate frontend config dynamically from client config
- Use IAM roles with least privilege
- Rotate secrets regularly

### ❌ DON'T
- Put sensitive credentials in frontend code
- Hardcode credentials in Lambda functions
- Store secrets in environment variables
- Use the same credentials across environments
- Commit secrets to version control
- Share secret keys between services
